# 📊 Análisis de Churn en SQL

---

## 🌟 Introducción

✨ Este proyecto demuestra cómo realizar un **análisis de churn (abandono de clientes)** utilizando **SQL**, con el objetivo de identificar cuándo y por qué los clientes dejan de interactuar o generar valor.

El churn es una de las métricas más críticas en cualquier negocio, ya que impacta directamente en los ingresos y el crecimiento.

---

## 🎯 Objetivo del Proyecto

🚀 Detectar clientes inactivos y generar **insights accionables** que permitan:

* 📉 Reducir la pérdida de clientes
* 🎯 Mejorar estrategias de retención
* 💰 Incrementar el valor del cliente (LTV)

---

## 🧠 ¿Qué es Churn?

📊 El churn representa el porcentaje de clientes que dejan de utilizar un servicio o dejan de comprar en un periodo determinado.

En este proyecto:

* Cliente activo = tiene actividad reciente
* Cliente churn = no ha tenido actividad en un periodo definido

---

## 🗄️ Estructura de los Datos

El dataset incluye:

* 📅 Fecha
* 🧑‍💼 Cliente
* 💳 Producto
* 🔢 Transacciones
* 💰 TPV
* 📊 Utilidad

---

## ⚙️ Metodología

El análisis se construye en 4 pasos:

1. 🥇 Identificar la última actividad de cada cliente
2. ⏳ Calcular días de inactividad
3. 🚨 Definir umbral de churn (ej: 30 días)
4. 📊 Clasificar clientes como activos o churn

---

## 📊 Clasificación de Clientes

* 🟢 **Activos:** actividad reciente
* 🟡 **En riesgo:** inactividad moderada
* 🔴 **Churn:** alta inactividad

---

## 📈 Resultado Esperado

Obtendrás:

* Número de clientes activos vs churn
* Tasa de churn
* Identificación de clientes en riesgo

---

## 🔥 Valor de Negocio

Este análisis permite:

* 📉 Reducir abandono de clientes
* 🎯 Crear campañas de retención
* 📊 Mejorar la experiencia del cliente
